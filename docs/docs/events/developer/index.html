Developer WebSocket Events

Developer applications can subscribe to real-time events via WebSocket to receive notifications about platform activity.

Overview​

The Developer WebSocket namespace (/developer) allows applications to:

  • Connect using their application's websocketId
  • Subscribe to specific events based on their granted scopes
  • Receive real-time event notifications
  • Filter events by scope permissions

Connection​

Connect to the Developer WebSocket namespace:

import { io } from 'socket.io-client';

const socket = io('https://api.metaenix.com/developer', {
  query: {
    websocketId: 'your-app-websocket-id'
  }
});

socket.on('connected', (data) => {
  console.log('Connected:', data);
  // {
  //   message: 'Connected to developer gateway',
  //   websocketId: '...',
  //   applicationId: '...',
  //   applicationName: 'My App',
  //   scopes: ['read:profile', 'read:posts']
  // }
});

socket.on('error', (error) => {
  console.error('Error:', error.message);
});

Authentication​

Applications authenticate using their websocketId (UUID), which is generated when the application is created. You can find your websocketId in your application details.

Connection Methods:

  1. Query Parameter:

    io('https://api.metaenix.com/developer', {
      query: { websocketId: 'your-websocket-id' }
    });

  2. Header:

    io('https://api.metaenix.com/developer', {
      extraHeaders: {
        'x-websocket-id': 'your-websocket-id'
      }
    });

  3. Auth Object:

    io('https://api.metaenix.com/developer', {
      auth: {
        websocketId: 'your-websocket-id'
      }
    });

Subscribing to Events​

Subscribe to events your application has permission for:

Subscribe to Single Event​

socket.emit('subscribe', {
  event: 'user.followed'
});

socket.on('subscribed', (data) => {
  console.log('Subscribed:', data.subscribed);
  // ['user.followed']
  
  if (data.rejected) {
    console.log('Rejected:', data.rejected);
    // [{ event: 'post.created', reason: 'Missing required scope: read:posts' }]
  }
});

Subscribe to Multiple Events​

socket.emit('subscribe', {
  events: ['user.followed', 'post.created', 'post.liked']
});

socket.on('subscribed', (data) => {
  console.log('Subscribed to:', data.subscribed);
  console.log('Rejected:', data.rejected);
});

Unsubscribing from Events​

Unsubscribe from events you no longer need:

socket.emit('unsubscribe', {
  event: 'user.followed'
});

// Or multiple events
socket.emit('unsubscribe', {
  events: ['post.created', 'post.liked']
});

socket.on('unsubscribed', (data) => {
  console.log('Unsubscribed from:', data.unsubscribed);
});

Listing Subscriptions​

Get a list of your current subscriptions:

socket.emit('list_subscriptions');

socket.on('subscriptions', (data) => {
  console.log('Subscribed events:', data.events);
  console.log('Your scopes:', data.scopes);
});

Receiving Events​

All subscribed events are delivered via the event message:

socket.on('event', (payload) => {
  console.log('Event received:', payload.type);
  console.log('Data:', payload.data);
  
  // Handle specific event types
  switch (payload.type) {
    case 'user.followed':
      handleUserFollowed(payload.data);
      break;
    case 'post.created':
      handlePostCreated(payload.data);
      break;
    // ... other events
  }
});

Complete Example​

import { io } from 'socket.io-client';

const WEBSOCKET_ID = 'your-app-websocket-id';

const socket = io('https://api.metaenix.com/developer', {
  query: { websocketId: WEBSOCKET_ID }
});

// Connection handling
socket.on('connected', (data) => {
  console.log('Connected as:', data.applicationName);
  
  // Subscribe to events
  socket.emit('subscribe', {
    events: ['user.followed', 'post.created', 'post.liked']
  });
});

socket.on('subscribed', (data) => {
  console.log('Successfully subscribed to:', data.subscribed);
  
  if (data.rejected && data.rejected.length > 0) {
    console.warn('Some subscriptions were rejected:', data.rejected);
  }
});

// Receive events
socket.on('event', (payload) => {
  console.log(`[${payload.type}]`, payload.data);
  
  // Handle specific events
  if (payload.type === 'post.created') {
    // Notify users, update UI, etc.
    handleNewPost(payload.data);
  }
});

socket.on('error', (error) => {
  console.error('WebSocket error:', error.message);
});

socket.on('disconnect', () => {
  console.log('Disconnected from developer gateway');
});

// Handle reconnection
socket.on('reconnect', () => {
  console.log('Reconnected, resubscribing...');
  socket.emit('subscribe', {
    events: ['user.followed', 'post.created', 'post.liked']
  });
});

Error Handling​

Authentication Errors​

socket.on('error', (error) => {
  if (error.message.includes('Authentication failed')) {
    // Check websocketId
    // Verify application is active
  }
});

Subscription Errors​

When subscribing, check the rejected array:

socket.on('subscribed', (data) => {
  if (data.rejected) {
    data.rejected.forEach(({ event, reason }) => {
      console.error(`Cannot subscribe to ${event}: ${reason}`);
      // Usually means missing scope - request scope in app settings
    });
  }
});

Best Practices​

  1. Reconnection: Always handle reconnection and resubscribe to events
  2. Error Handling: Monitor error events and handle gracefully
  3. Scope Management: Only subscribe to events you have scopes for
  4. Rate Limiting: Be aware of rate limits on your application
  5. Connection Management: Disconnect when not needed to save resources
  6. Event Filtering: Filter events client-side if needed for your use case

Security​

  • websocketId is Secret: Treat your websocketId like a secret - don't expose it in client-side code
  • Scope Validation: Events are filtered by scope server-side - you'll only receive events you're authorized for
  • Application Status: Only active applications can connect
  • Single Connection: Only one connection per application is allowed (new connections disconnect old ones)
