Event Payloads

Complete reference for all event types and their payload structures.

User Events​

user.followed​

Emitted when a user follows another user.

Namespace: /follows

Event Name: new_follower

Payload:

{
  "followerId": "123e4567-e89b-12d3-a456-426614174000",
  "timestamp": "2025-01-15T10:00:00.000Z"
}

Fields:

FieldTypeDescription
followerIdstring (UUID)ID of the user who followed
timestampstring (ISO 8601)When the follow occurred

Example:

socket.on('new_follower', (data) => {
  console.log(`User ${data.followerId} followed you at ${data.timestamp}`);
});

Connection Events​

connected​

Emitted when a WebSocket connection is successfully established and authenticated.

Namespace: /account, /follows, /posts

Event Name: connected

Payload:

{
  "message": "Connected to account gateway",
  "websocketId": "123e4567-e89b-12d3-a456-426614174000",
  "userId": "123e4567-e89b-12d3-a456-426614174001",
  "username": "johndoe"
}

Fields:

FieldTypeDescription
messagestringConnection success message
websocketIdstring (UUID)Your application's websocket ID
userIdstring (UUID)User ID if user-authenticated connection
usernamestringUsername if user-authenticated connection

error​

Emitted when an error occurs.

Namespace: All namespaces

Event Name: error

Payload:

{
  "message": "Connection failed: Invalid websocketId"
}

Common Error Messages:

  • "Connection failed: Invalid websocketId" - Invalid or missing websocketId
  • "Connection failed: Internal server error" - Server-side error
  • "New connection established from another location" - Another connection replaced this one
  • "Unauthorized" - Not authorized to access this namespace

subscribed​

Emitted when successfully subscribed to an event type.

Namespace: /follows

Event Name: subscribed (response to subscribe_follows)

Payload:

{
  "event": "subscribed",
  "room": "user:123e4567-e89b-12d3-a456-426614174001"
}

Fields:

FieldTypeDescription
eventstringAlways "subscribed"
roomstringThe room identifier you've joined

Payload Validation​

Always validate event payloads before processing:

function validateNewFollowerPayload(data) {
  if (!data || typeof data !== 'object') {
    throw new Error('Invalid payload: must be an object');
  }
  
  if (!data.followerId || typeof data.followerId !== 'string') {
    throw new Error('Invalid payload: followerId must be a string');
  }
  
  if (!data.timestamp || typeof data.timestamp !== 'string') {
    throw new Error('Invalid payload: timestamp must be a string');
  }
  
  // Validate UUID format
  const uuidRegex = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;
  if (!uuidRegex.test(data.followerId)) {
    throw new Error('Invalid payload: followerId must be a valid UUID');
  }
  
  // Validate ISO 8601 timestamp
  if (isNaN(Date.parse(data.timestamp))) {
    throw new Error('Invalid payload: timestamp must be a valid ISO 8601 date');
  }
  
  return true;
}

socket.on('new_follower', (data) => {
  try {
    validateNewFollowerPayload(data);
    // Process valid event
  } catch (error) {
    console.error('Invalid event payload:', error);
  }
});

TypeScript Types​

For TypeScript projects, define types for event payloads:

interface NewFollowerPayload {
  followerId: string;
  timestamp: string;
}

interface UnfollowPayload {
  followerId: string;
  timestamp: string;
}

interface ConnectedPayload {
  message: string;
  websocketId: string;
  userId?: string;
  username?: string;
}

interface ErrorPayload {
  message: string;
}

// Usage
socket.on('new_follower', (data: NewFollowerPayload) => {
  console.log(data.followerId);
});

Event Ordering​

Events are delivered in the order they occur, but network conditions may cause slight delays. Always use the timestamp field to determine the actual order of events if ordering is critical.

Event Deduplication​

Events may be delivered multiple times due to reconnections. Implement idempotency using event IDs or timestamps:

const processedEvents = new Set();

socket.on('new_follower', (data) => {
  const eventId = `${data.followerId}-${data.timestamp}`;
  
  if (processedEvents.has(eventId)) {
    console.log('Duplicate event ignored:', eventId);
    return;
  }
  
  processedEvents.add(eventId);
  // Process event
});
