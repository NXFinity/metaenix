Events Introduction

Meta EN|IX provides a real-time event system that allows your application to subscribe to and receive notifications about various platform events as they happen.

Overview​

The Events API uses WebSockets to deliver real-time notifications to your application. When events occur in the platform (such as a user being followed, a post being created, or a comment being added), your application can receive instant notifications without polling.

How It Works​

  1. Connect - Establish a WebSocket connection to the Meta EN|IX event server
  2. Authenticate - Authenticate using your application's credentials
  3. Subscribe - Subscribe to specific event types you're interested in
  4. Receive - Receive real-time event notifications as they occur

Event Types​

Events are organized by category:

  • User Events - User-related actions (follows, profile updates)
  • Post Events - Content-related actions (posts created, comments added)
  • Interaction Events - User interactions (likes, shares, bookmarks)

WebSocket Namespaces​

Events are delivered through different WebSocket namespaces:

  • /account - Account-level events and notifications
  • /follows - Follow/unfollow events
  • /posts - Post and content-related events

Authentication​

All WebSocket connections require authentication using your application's websocketId. This is obtained when you register your application and is unique to your app.

Benefits​

  • Real-time Updates - Receive events instantly as they happen
  • Efficient - No need to poll endpoints repeatedly
  • Scalable - WebSocket connections are lightweight and efficient
  • Reliable - Built on Socket.IO with automatic reconnection
