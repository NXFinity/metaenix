OAuth Authentication

Meta EN|IX uses OAuth 2.0 for authentication and authorization. This guide explains how to implement OAuth in your application.

OAuth Flow Overview​

The OAuth 2.0 authorization code flow consists of these steps:

  1. Redirect user to authorization URL - User authorizes your application
  2. Receive authorization code - User is redirected back with a code
  3. Exchange code for access token - Server-side exchange using your client secret
  4. Use access token - Make API requests with the access token

Step 1: Redirect User to Authorization URL​

Construct the authorization URL with the following parameters:

https://api.metaenix.com/v1/oauth/authorize?
  client_id=YOUR_CLIENT_ID
  &redirect_uri=YOUR_REDIRECT_URI
  &response_type=code
  &scope=SCOPES
  &state=RANDOM_STATE_STRING

Parameters​

ParameterRequiredDescription
client_idYesYour application's Client ID
redirect_uriYesMust match a registered redirect URI
response_typeYesMust be code
scopeYesSpace-separated list of scopes (see Scopes)
stateRecommendedRandom string to prevent CSRF attacks

Example Authorization URL​

const clientId = 'your-client-id';
const redirectUri = encodeURIComponent('https://yourapp.com/callback');
const scopes = encodeURIComponent('read:user write:post');
const state = generateRandomString(); // Store in session

const authUrl = `https://api.metaenix.com/v1/oauth/authorize?` +
  `client_id=${clientId}&` +
  `redirect_uri=${redirectUri}&` +
  `response_type=code&` +
  `scope=${scopes}&` +
  `state=${state}`;

// Redirect user to authUrl
window.location.href = authUrl;

Step 2: Handle Authorization Callback​

After the user authorizes your application, they'll be redirected to your redirect_uri with a code and state parameter:

https://yourapp.com/callback?code=AUTHORIZATION_CODE&state=YOUR_STATE

Validate the State​

Always validate that the state parameter matches what you sent:

// In your callback handler
const { code, state } = req.query;

// Verify state matches what you stored
if (state !== storedState) {
  throw new Error('Invalid state parameter');
}

Step 3: Exchange Code for Access Token​

Exchange the authorization code for an access token on your server:

POST /v1/oauth/token
Content-Type: application/x-www-form-urlencoded

Request Body​

grant_type=authorization_code
&code=AUTHORIZATION_CODE
&redirect_uri=YOUR_REDIRECT_URI
&client_id=YOUR_CLIENT_ID
&client_secret=YOUR_CLIENT_SECRET

Example Request​

const response = await fetch('https://api.metaenix.com/v1/oauth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: new URLSearchParams({
    grant_type: 'authorization_code',
    code: authorizationCode,
    redirect_uri: redirectUri,
    client_id: clientId,
    client_secret: clientSecret, // Keep this server-side!
  }),
});

const data = await response.json();

Response​

{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "refresh_token_here",
  "scope": "read:user write:post"
}

Step 4: Use Access Token​

Include the access token in the Authorization header for all API requests:

const response = await fetch('https://api.metaenix.com/v1/users/me', {
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json',
  },
});

const userData = await response.json();

Refresh Tokens​

Access tokens expire after a set period. Use the refresh token to obtain a new access token:

POST /v1/oauth/token
Content-Type: application/x-www-form-urlencoded

Request Body​

grant_type=refresh_token
&refresh_token=REFRESH_TOKEN
&client_id=YOUR_CLIENT_ID
&client_secret=YOUR_CLIENT_SECRET

Example​

const response = await fetch('https://api.metaenix.com/v1/oauth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: new URLSearchParams({
    grant_type: 'refresh_token',
    refresh_token: refreshToken,
    client_id: clientId,
    client_secret: clientSecret,
  }),
});

const { access_token, refresh_token } = await response.json();

Complete Example​

Here's a complete example using Node.js and Express:

const express = require('express');
const app = express();

const CLIENT_ID = process.env.CLIENT_ID;
const CLIENT_SECRET = process.env.CLIENT_SECRET;
const REDIRECT_URI = 'https://yourapp.com/callback';

// Step 1: Redirect to authorization
app.get('/auth', (req, res) => {
  const state = generateRandomString();
  req.session.state = state;
  
  const authUrl = `https://api.metaenix.com/v1/oauth/authorize?` +
    `client_id=${CLIENT_ID}&` +
    `redirect_uri=${encodeURIComponent(REDIRECT_URI)}&` +
    `response_type=code&` +
    `scope=read:user write:post&` +
    `state=${state}`;
  
  res.redirect(authUrl);
});

// Step 2 & 3: Handle callback and exchange code
app.get('/callback', async (req, res) => {
  const { code, state } = req.query;
  
  // Validate state
  if (state !== req.session.state) {
    return res.status(400).send('Invalid state');
  }
  
  // Exchange code for token
  const tokenResponse = await fetch('https://api.metaenix.com/v1/oauth/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      grant_type: 'authorization_code',
      code,
      redirect_uri: REDIRECT_URI,
      client_id: CLIENT_ID,
      client_secret: CLIENT_SECRET,
    }),
  });
  
  const tokens = await tokenResponse.json();
  
  // Store tokens securely (e.g., in session or database)
  req.session.accessToken = tokens.access_token;
  req.session.refreshToken = tokens.refresh_token;
  
  res.redirect('/dashboard');
});

// Step 4: Make API calls
app.get('/api/user', async (req, res) => {
  const response = await fetch('https://api.metaenix.com/v1/users/me', {
    headers: {
      'Authorization': `Bearer ${req.session.accessToken}`,
    },
  });
  
  const userData = await response.json();
  res.json(userData);
});

Error Handling​

The OAuth endpoints may return errors:

{
  "error": "invalid_request",
  "error_description": "The request is missing a required parameter"
}

Common errors:

  • invalid_request - Missing or invalid parameters
  • invalid_client - Invalid client credentials
  • invalid_grant - Invalid authorization code or refresh token
  • unauthorized_client - Client not authorized for this grant type
  • invalid_scope - Requested scope is invalid

Always handle errors gracefully and provide user feedback.

Security Considerations​

  1. Never expose client secret - Keep it server-side only
  2. Use HTTPS - Always use HTTPS in production
  3. Validate state - Always validate the state parameter
  4. Store tokens securely - Use secure storage (encrypted database, secure cookies)
  5. Handle token expiration - Implement token refresh logic
  6. Use minimal scopes - Only request the scopes you need
