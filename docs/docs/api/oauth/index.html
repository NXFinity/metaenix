OAuth API

The OAuth API provides endpoints for OAuth 2.0 authentication and authorization flows.

Base Endpoint​

GET|POST /v1/oauth/*

Authorization Endpoint​

Generate an authorization code for the OAuth 2.0 Authorization Code flow.

Endpoint​

GET /v1/oauth/authorize

Requires: User authentication (session-based, not OAuth token)

Query Parameters​

ParameterRequiredDescription
client_idYesYour application's Client ID
redirect_uriYesMust match a registered redirect URI
response_typeYesMust be code
scopeYesSpace-separated list of scopes
stateRecommendedRandom string for CSRF protection
code_challengeOptionalPKCE code challenge (for public clients)
code_challenge_methodOptionalPKCE method: S256 or plain

Example Request​

const clientId = 'your-client-id';
const redirectUri = encodeURIComponent('https://myapp.com/callback');
const scopes = encodeURIComponent('read:profile write:posts');
const state = generateRandomString();

const authUrl = `https://api.metaenix.com/v1/oauth/authorize?` +
  `client_id=${clientId}&` +
  `redirect_uri=${redirectUri}&` +
  `response_type=code&` +
  `scope=${scopes}&` +
  `state=${state}`;

// Redirect user to authUrl
window.location.href = authUrl;

Response​

User is redirected to your redirect_uri with:

https://myapp.com/callback?code=AUTHORIZATION_CODE&state=YOUR_STATE

Error Responses​

400 Bad Request - Invalid parameters:

{
  "statusCode": 400,
  "message": "Invalid redirect URI"
}

404 Not Found - Application not found:

{
  "statusCode": 404,
  "message": "Application not found"
}

403 Forbidden - Application not active:

{
  "statusCode": 403,
  "message": "Application is not active"
}

Token Endpoint​

Exchange authorization code for access token, refresh access token, or issue client credentials token.

Endpoint​

POST /v1/oauth/token

Public endpoint - No authentication required (credentials in body)

Request Body​

Content-Type: application/x-www-form-urlencoded

Authorization Code Grant​

grant_type=authorization_code
&code=AUTHORIZATION_CODE
&redirect_uri=YOUR_REDIRECT_URI
&client_id=YOUR_CLIENT_ID
&client_secret=YOUR_CLIENT_SECRET
&code_verifier=PKCE_CODE_VERIFIER (if using PKCE)

Refresh Token Grant​

grant_type=refresh_token
&refresh_token=REFRESH_TOKEN
&client_id=YOUR_CLIENT_ID
&client_secret=YOUR_CLIENT_SECRET

Client Credentials Grant​

grant_type=client_credentials
&client_id=YOUR_CLIENT_ID
&client_secret=YOUR_CLIENT_SECRET
&scope=read:profile read:posts (optional)

Example Request​

// Authorization Code Grant
const response = await fetch('https://api.metaenix.com/v1/oauth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: new URLSearchParams({
    grant_type: 'authorization_code',
    code: authorizationCode,
    redirect_uri: redirectUri,
    client_id: clientId,
    client_secret: clientSecret,
    code_verifier: codeVerifier, // If using PKCE
  }),
});

const tokens = await response.json();

Response​

{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "tokenType": "Bearer",
  "expiresIn": 3600,
  "scope": "read:profile write:posts"
}

Note: For client credentials grant, refreshToken will be null.

Error Responses​

400 Bad Request - Invalid grant:

{
  "statusCode": 400,
  "message": "Invalid authorization code"
}

401 Unauthorized - Invalid credentials:

{
  "statusCode": 401,
  "message": "Invalid client credentials"
}

403 Forbidden - Application not active:

{
  "statusCode": 403,
  "message": "Application is not active"
}

Revocation Endpoint​

Revoke an access token or refresh token.

Endpoint​

POST /v1/oauth/revoke

Public endpoint - No authentication required

Request Body​

Content-Type: application/x-www-form-urlencoded

token=TOKEN_TO_REVOKE
&token_type_hint=access_token (optional: access_token or refresh_token)

Example Request​

const response = await fetch('https://api.metaenix.com/v1/oauth/revoke', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: new URLSearchParams({
    token: accessToken,
    token_type_hint: 'access_token',
  }),
});

// 200 OK on success

Response​

200 OK - Token revoked successfully:

{
  "message": "Token revoked successfully"
}

Error Responses​

400 Bad Request - Invalid token:

{
  "statusCode": 400,
  "message": "Invalid token"
}

Introspection Endpoint​

Get information about a token (RFC 7662).

Endpoint​

POST /v1/oauth/introspect

Public endpoint - No authentication required

Request Body​

Content-Type: application/x-www-form-urlencoded

token=TOKEN_TO_INTROSPECT
&token_type_hint=access_token (optional: access_token or refresh_token)

Example Request​

const response = await fetch('https://api.metaenix.com/v1/oauth/introspect', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: new URLSearchParams({
    token: accessToken,
    token_type_hint: 'access_token',
  }),
});

const introspection = await response.json();

Response​

Active Token:

{
  "active": true,
  "scope": "read:profile write:posts",
  "clientId": "abc123def456",
  "username": "johndoe",
  "exp": 1736942400
}

Inactive/Invalid Token:

{
  "active": false
}

Response Fields​

FieldTypeDescription
activebooleanWhether the token is active
scopestringSpace-separated list of scopes
clientIdstringApplication client ID
usernamestringUsername (null for client credentials)
expnumberExpiration timestamp (Unix)

Scopes Endpoint​

List all available OAuth scopes with their descriptions and metadata.

Endpoint​

GET /v1/oauth/scopes

Public endpoint - No authentication required

Example Request​

const response = await fetch('https://api.metaenix.com/v1/oauth/scopes');

const { scopes } = await response.json();

Response​

{
  "scopes": [
    {
      "id": "read:profile",
      "name": "Read Profile",
      "description": "Read user profile information",
      "category": "read",
      "group": "profile",
      "requiresApproval": false,
      "isDefault": true
    },
    {
      "id": "write:posts",

      "description": "Create, update, and delete posts",
      "category": "write",
      "group": "posts",
      "requiresApproval": true,
      "isDefault": false

  ]
}

Grant Types​

Authorization Code Flow​

Best for web applications and mobile apps. User authorizes your app, you receive an authorization code, then exchange it for tokens server-side.

Flow:

  1. Redirect user to /oauth/authorize
  2. User authorizes your app
  3. Receive authorization code via redirect
  4. Exchange code for tokens at /oauth/token

Refresh Token Flow​

Use refresh tokens to obtain new access tokens when they expire.

Flow:

  1. Use refresh token at /oauth/token with grant_type=refresh_token
  2. Receive new access token and refresh token

Client Credentials Flow​

For server-to-server communication without user interaction.

Flow:

  1. Request tokens at /oauth/token with grant_type=client_credentials
  2. Receive access token (no refresh token)

PKCE (Proof Key for Code Exchange)​

PKCE is recommended for public clients (mobile apps, single-page apps) to prevent authorization code interception.

Flow:

  1. Generate code_verifier (random string)
  2. Generate code_challenge from code_verifier:
    • Method S256: SHA256 hash, base64url encoded
    • Method plain: code_verifier as-is
  3. Include code_challenge and code_challenge_method in authorization request
  4. Include code_verifier when exchanging code for token

Example PKCE Implementation​

// Generate code verifier
const codeVerifier = generateRandomString(128);

// Generate code challenge (S256)
const encoder = new TextEncoder();
const data = encoder.encode(codeVerifier);
const hash = await crypto.subtle.digest('SHA-256', data);
const codeChallenge = base64UrlEncode(hash);

// Authorization request
const authUrl = `https://api.metaenix.com/v1/oauth/authorize?` +
  `client_id=${clientId}&` +
  `redirect_uri=${redirectUri}&` +
  `response_type=code&` +
  `scope=${scopes}&` +
  `code_challenge=${codeChallenge}&` +
  `code_challenge_method=S256`;

// Token exchange
const tokenResponse = await fetch('https://api.metaenix.com/v1/oauth/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  body: new URLSearchParams({
    grant_type: 'authorization_code',
    code: authorizationCode,
    redirect_uri: redirectUri,
    client_id: clientId,
    code_verifier: codeVerifier, // Include verifier
  }),
});

Token Usage​

Include the access token in the Authorization header for all API requests:

const response = await fetch('https://api.metaenix.com/v1/users/me', {
  headers: {
    'Authorization': `Bearer ${accessToken}`,
  },
});

Error Handling​

OAuth endpoints return standard OAuth 2.0 error responses:

{
  "error": "invalid_request",
  "error_description": "The request is missing a required parameter"
}

Common Errors​

  • invalid_request - Missing or invalid parameters
  • invalid_client - Invalid client credentials
  • invalid_grant - Invalid authorization code or refresh token
  • unauthorized_client - Client not authorized for this grant type
  • invalid_scope - Requested scope is invalid or not approved
  • access_denied - User denied authorization

Security Best Practices​

  1. Never expose client secret - Keep it server-side only
  2. Use HTTPS - Always use HTTPS in production
  3. Validate state - Always validate the state parameter
  4. Store tokens securely - Use secure storage (encrypted database, secure cookies)
  5. Handle token expiration - Implement token refresh logic
  6. Use minimal scopes - Only request the scopes you need
  7. Use PKCE - Always use PKCE for public clients
