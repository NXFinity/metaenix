Developer API

The Developer API allows you to manage your developer account and applications.

Base Endpoint​

GET|POST|PATCH|DELETE /v1/developer/*

Authentication​

All Developer API endpoints require authentication using a session-based token (not OAuth tokens). You must be logged in to your Meta EN|IX account.

Check Developer Status​

Check if you meet the requirements to become a developer and see your current status.

Endpoint​

GET /v1/developer/status

Headers​

Authorization: Bearer YOUR_SESSION_TOKEN

Example Request​

const response = await fetch('https://api.metaenix.com/v1/developer/status', {
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
  },
});

const status = await response.json();

Response​

{
  "eligible": false,
  "requirements": {
    "profileComplete": true,
    "emailVerified": true,
    "twoFactorEnabled": false,
    "accountAge": true,
    "accountStanding": true,
    "accountActivity": true
  },
  "errors": [
    "Two-factor authentication must be enabled"
  ]
}

Requirements​

To become a developer, you must meet all of these requirements:

  • ✅ Profile Complete - Username and email must be set
  • ✅ Email Verified - Email address must be verified
  • ✅ Two-Factor Authentication - 2FA must be enabled
  • ✅ Account Age - Account must be at least 30 days old
  • ✅ Account Standing - Account must be in good standing (not banned or timed out)
  • ✅ Account Activity - Account must show recent activity

Register as Developer​

Register your account as a developer. This allows you to create applications.

Endpoint​

POST /v1/developer/register

Request Body​

{
  "acceptTerms": true
}

Example Request​

const response = await fetch('https://api.metaenix.com/v1/developer/register', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    acceptTerms: true,
  }),
});

const result = await response.json();

Response​

{
  "id": "123e4567-e89b-12d3-a456-426614174000",
  "username": "johndoe",
  "email": "john@example.com",
  "isDeveloper": true,
  "developerTermsAcceptedAt": "2025-01-15T10:00:00.000Z"
}

Error Responses​

400 Bad Request - Requirements not met:

{
  "statusCode": 400,
  "message": "Developer registration requirements not met",
  "requirements": {
    "profileComplete": true,
    "emailVerified": true,
    "twoFactorEnabled": false,
    "accountAge": true,
    "accountStanding": true,
    "accountActivity": true
  },
  "errors": [
    "Two-factor authentication must be enabled"
  ]
}

400 Bad Request - Already a developer:

{
  "statusCode": 400,
  "message": "User is already a developer"
}

GET /v1/developer/apps

const response = await fetch('https://api.metaenix.com/v1/developer/apps', {
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
  },
});

const apps = await response.json();

    "rateLimit": 1000,
    "dateCreated": "2025-01-15T10:00:00.000Z"

Create Application​

Create a new application. Maximum 2 applications per developer (1 Production + 1 Development).

Endpoint​

POST /v1/developer/apps

Request Body​

{
  "name": "My Awesome App",
  "description": "A cool application that does amazing things",
  "environment": "DEVELOPMENT",
  "redirectUris": [
    "https://myapp.com/callback",
    "http://localhost:3000/callback"
  ],
  "iconUrl": "https://myapp.com/icon.png",
  "websiteUrl": "https://myapp.com",
  "privacyPolicyUrl": "https://myapp.com/privacy",
  "termsOfServiceUrl": "https://myapp.com/terms",
  "scopes": [
    "read:profile",
    "read:posts",
    "write:posts"
  ]
}

Example Request​

const response = await fetch('https://api.metaenix.com/v1/developer/apps', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'My Awesome App',
    description: 'A cool application',
    environment: 'DEVELOPMENT',
    redirectUris: ['https://myapp.com/callback'],
    scopes: ['read:profile', 'read:posts'],
  }),
});

const result = await response.json();

Response​

{
  "application": {
    "id": "app-123",
    "name": "My Awesome App",
    "clientId": "abc123def456",
    "environment": "DEVELOPMENT",
    "status": "ACTIVE",
    "rateLimit": 1000,
    "dateCreated": "2025-01-15T10:00:00.000Z"
  },
  "clientSecret": "your-client-secret-here-KEEP-THIS-SECRET"
}

⚠️ Important: The clientSecret is shown only once. Store it securely immediately. You cannot retrieve it later.

Error Responses​

400 Bad Request - Application limit reached:

{
  "statusCode": 400,
  "message": "Maximum application limit reached. You can have a maximum of 2 applications (1 Production + 1 Development)."
}

400 Bad Request - Environment already exists:

{
  "statusCode": 400,
  "message": "You already have a DEVELOPMENT application. Maximum 1 app per environment."
}

403 Forbidden - Not a developer:

{
  "statusCode": 403,
  "message": "User is not a developer"
}

Get Application Details​

Get details of a specific application.

Endpoint​

GET /v1/developer/apps/:id

Path Parameters​

ParameterTypeDescription

Example Request​

const appId = 'app-123';
const response = await fetch(`https://api.metaenix.com/v1/developer/apps/${appId}`, {
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
  },
});

Response​

Same format as application object in List Applications.

Error Responses​

404 Not Found - Application not found:

{
  "statusCode": 404,
  "message": "Application not found"
}

403 Forbidden - Access denied:

{
  "statusCode": 403,
  "message": "Access denied"
}

Update Application​

Update application details. Cannot change environment. Production apps in PENDING status can be updated before approval.

Endpoint​

PATCH /v1/developer/apps/:id

Path Parameters​

ParameterTypeDescription
idstringApplication ID

Request Body​

All fields are optional:

{
  "name": "Updated App Name",
  "description": "Updated description",
  "redirectUris": [
    "https://myapp.com/callback",
    "https://myapp.com/new-callback"
  ],
  "iconUrl": "https://myapp.com/new-icon.png",
  "websiteUrl": "https://myapp.com",
  "privacyPolicyUrl": "https://myapp.com/privacy",
  "termsOfServiceUrl": "https://myapp.com/terms",
  "scopes": [
    "read:profile",
    "read:posts",
    "write:posts",
    "read:comments"
  ]
}

Example Request​

const appId = 'app-123';
const response = await fetch(`https://api.metaenix.com/v1/developer/apps/${appId}`, {
  method: 'PATCH',
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Updated App Name',
    redirectUris: ['https://myapp.com/new-callback'],
  }),
});

const app = await response.json();

Response​

Updated application object.

Error Responses​

400 Bad Request - Cannot change environment:

{
  "statusCode": 400,
  "message": "Cannot change application environment"
}

403 Forbidden - Access denied or cannot update approved production app:

{
  "statusCode": 403,
  "message": "Cannot update approved production application"
}

Delete Application​

Delete an application. After deletion, you can create a new one (still limited to max 2 total).

Endpoint​

DELETE /v1/developer/apps/:id

Path Parameters​

ParameterTypeDescription
idstringApplication ID

Example Request​

const appId = 'app-123';
const response = await fetch(`https://api.metaenix.com/v1/developer/apps/${appId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
  },
});

// 204 No Content on success

Response​

204 No Content - Application deleted successfully.

Error Responses​

404 Not Found - Application not found:

{
  "statusCode": 404,
  "message": "Application not found"
}

403 Forbidden - Access denied:

{
  "statusCode": 403,
  "message": "Access denied"
}

Regenerate Client Secret​

Regenerate the client secret for an application. The old secret will be invalidated immediately.

Endpoint​

POST /v1/developer/apps/:id/regenerate-secret

Path Parameters​

ParameterTypeDescription
idstringApplication ID

Example Request​

const appId = 'app-123';
const response = await fetch(`https://api.metaenix.com/v1/developer/apps/${appId}/regenerate-secret`, {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
  },
});

const result = await response.json();

Response​

{
  "clientSecret": "new-client-secret-here-KEEP-THIS-SECRET"
}

⚠️ Important: The new clientSecret is shown only once. Store it securely immediately. The old secret is invalidated immediately.

Error Responses​

404 Not Found - Application not found:

{
  "statusCode": 404,
  "message": "Application not found"
}

403 Forbidden - Access denied:

{
  "statusCode": 403,
  "message": "Access denied"
}

Application Status​

Applications can have the following statuses:

  • PENDING - Production application awaiting admin approval
  • ACTIVE - Application is active and can be used
  • SUSPENDED - Application is temporarily suspended
  • REVOKED - Application access has been revoked
  • REJECTED - Production application was rejected by admin

Development applications are automatically set to ACTIVE status. Production applications start as PENDING and require admin approval.

Rate Limits​

Applications have different rate limits based on environment:

  • Development Apps: 1,000 requests/hour per app+user combination
  • Production Apps: 10,000 requests/hour per app+user combination

Rate limits are enforced per application and user combination, ensuring fair usage distribution.
