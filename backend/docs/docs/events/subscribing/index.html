Subscribing to Events

Learn how to connect to the Meta EN|IX WebSocket server and subscribe to events.

Prerequisites​

Before subscribing to events, you need:

  1. Application Credentials - Your clientId and clientSecret from app registration
  2. WebSocket ID - Your application's unique websocketId (obtained during app registration)
  3. WebSocket Client - A Socket.IO client library for your platform

WebSocket Endpoint​

Connect to the Meta EN|IX WebSocket server:

wss://api.metaenix.com

Connection Methods​

Method 1: Query Parameter​

Include your websocketId as a query parameter:

import { io } from 'socket.io-client';

const socket = io('https://api.metaenix.com/account', {
  query: {
    websocketId: 'your-websocket-id-here'
  },
  transports: ['websocket'],
  reconnection: true,
  reconnectionDelay: 1000,
  reconnectionAttempts: 5
});

Method 2: Authorization Header​

Include your websocketId in the authorization header:

const socket = io('https://api.metaenix.com/account', {
  auth: {
    websocketId: 'your-websocket-id-here'
  },
  transports: ['websocket'],
  reconnection: true
});

Method 3: Custom Header​

Include your websocketId in a custom header:

const socket = io('https://api.metaenix.com/account', {
  extraHeaders: {
    'x-websocket-id': 'your-websocket-id-here'
  },
  transports: ['websocket']
});

Connection Events​

Connection Successful​

When successfully connected, you'll receive a connected event:

socket.on('connected', (data) => {
  console.log('Connected:', data);
  // {
  //   message: 'Connected to account gateway',
  //   websocketId: 'your-websocket-id',
  //   userId: 'user-id-if-authenticated',
  //   username: 'username-if-authenticated'
  // }
});

Connection Error​

Handle connection errors:

socket.on('error', (error) => {
  console.error('Connection error:', error);
  // {
  //   message: 'Connection failed: Invalid websocketId'
  // }
});

socket.on('connect_error', (error) => {
  console.error('Connect error:', error.message);
});

Disconnection​

Handle disconnection events:

socket.on('disconnect', (reason) => {
  console.log('Disconnected:', reason);
  // Reasons: 'io server disconnect', 'io client disconnect', 
  //          'ping timeout', 'transport close', 'transport error'
});

Subscribing to Events​

Subscribe to Follow Events​

Connect to the /follows namespace and subscribe:

const followsSocket = io('https://api.metaenix.com/follows', {
  query: {
    websocketId: 'your-websocket-id-here'
  }
});

// Subscribe to follow events
followsSocket.emit('subscribe_follows', {}, (response) => {
  console.log('Subscribed:', response);
  // {
  //   event: 'subscribed',
  //   room: 'user:user-id'
  // }
});

Listen for Follow Events​

After subscribing, listen for events:

// New follower notification
followsSocket.on('new_follower', (data) => {
  console.log('New follower:', data);
  // {
  //   followerId: 'user-id-of-follower',
  //   timestamp: '2025-01-15T10:00:00Z'
  // }
});

// Unfollow notification
followsSocket.on('unfollow', (data) => {
  console.log('User unfollowed:', data);
  // {
  //   followerId: 'user-id-who-unfollowed',
  //   timestamp: '2025-01-15T10:00:00Z'
  // }
});

Complete Example​

Here's a complete example using Node.js:

import { io } from 'socket.io-client';

class MetaEnixEvents {
  constructor(websocketId) {
    this.websocketId = websocketId;
    this.sockets = {};
  }

  connect(namespace = 'account') {
    const socket = io(`https://api.metaenix.com/${namespace}`, {
      query: {
        websocketId: this.websocketId
      },
      transports: ['websocket'],
      reconnection: true,
      reconnectionDelay: 1000,
      reconnectionAttempts: 5
    });

    socket.on('connect', () => {
      console.log(`Connected to ${namespace} namespace`);
    });

    socket.on('connected', (data) => {
      console.log('Authenticated:', data);
    });

    socket.on('error', (error) => {
      console.error('Error:', error);
    });

    socket.on('disconnect', (reason) => {
      console.log('Disconnected:', reason);
    });

    this.sockets[namespace] = socket;
    return socket;
  }

  subscribeToFollows() {
    const socket = this.connect('follows');
    
    socket.emit('subscribe_follows', {}, (response) => {
      console.log('Subscribed to follows:', response);
    });

    socket.on('new_follower', (data) => {
      this.handleNewFollower(data);
    });

    socket.on('unfollow', (data) => {
      this.handleUnfollow(data);
    });

    return socket;
  }

  handleNewFollower(data) {
    console.log('New follower event:', data);
    // Implement your event handling logic here
  }

  handleUnfollow(data) {
    console.log('Unfollow event:', data);
    // Implement your event handling logic here
  }

  disconnect(namespace) {
    if (this.sockets[namespace]) {
      this.sockets[namespace].disconnect();
      delete this.sockets[namespace];
    }
  }

  disconnectAll() {
    Object.keys(this.sockets).forEach(namespace => {
      this.disconnect(namespace);
    });
  }
}

// Usage
const events = new MetaEnixEvents('your-websocket-id-here');
events.subscribeToFollows();

Reconnection Handling​

Socket.IO automatically handles reconnections. You can customize the behavior:

const socket = io('https://api.metaenix.com/account', {
  query: {
    websocketId: 'your-websocket-id-here'
  },
  reconnection: true,
  reconnectionDelay: 1000,        // Wait 1 second before reconnecting
  reconnectionDelayMax: 5000,     // Maximum delay between reconnection attempts
  reconnectionAttempts: 5,        // Maximum number of reconnection attempts
  timeout: 20000                   // Connection timeout
});

socket.on('reconnect', (attemptNumber) => {
  console.log(`Reconnected after ${attemptNumber} attempts`);
});

socket.on('reconnect_attempt', (attemptNumber) => {
  console.log(`Reconnection attempt ${attemptNumber}`);
});

socket.on('reconnect_failed', () => {
  console.error('Failed to reconnect after maximum attempts');
});

Error Handling​

Always implement proper error handling:

socket.on('error', (error) => {
  if (error.message.includes('Invalid websocketId')) {
    // Handle authentication error
    console.error('Authentication failed - check websocketId');
  } else if (error.message.includes('Unauthorized')) {
    // Handle authorization error
    console.error('Not authorized to access this namespace');
  } else {
    // Handle other errors
    console.error('Unexpected error:', error);
  }
});

Best Practices​

  1. Store websocketId Securely - Never expose your websocketId in client-side code
  2. Handle Reconnections - Implement reconnection logic for production
  3. Validate Events - Always validate event payloads before processing
  4. Rate Limiting - Be aware of rate limits on event subscriptions
  5. Error Handling - Implement comprehensive error handling
  6. Connection Management - Properly disconnect when done
