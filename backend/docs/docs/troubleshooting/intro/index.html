Troubleshooting

Common issues and solutions when integrating with the Meta EN|IX API.

Authentication Issues​

"Unauthorized" or 401 Errors​

Problem: Getting 401 Unauthorized errors when making API requests.

Possible Causes:

  • Missing or invalid access token
  • Expired access token
  • Incorrect token format

Solutions:

  1. Check Token Format
// ❌ BAD
headers: {
  'Authorization': `Token ${accessToken}`
}

// ✅ GOOD
headers: {
  'Authorization': `Bearer ${accessToken}`
}
  1. Verify Token Expiration
// Decode JWT token to check expiration
function isTokenExpired(token) {
  try {
    const payload = JSON.parse(atob(token.split('.')[1]));
    const expiration = payload.exp * 1000; // Convert to milliseconds
    return Date.now() >= expiration;
  } catch (error) {
    return true; // Assume expired if can't decode
  }
}

if (isTokenExpired(accessToken)) {
  await refreshToken();
}
  1. Check Token Scope
// Verify token has required scopes
function hasScope(token, requiredScope) {
  try {
    const payload = JSON.parse(atob(token.split('.')[1]));
    const scopes = payload.scope?.split(' ') || [];
    return scopes.includes(requiredScope);
  } catch {
    return false;
  }
}

OAuth Authorization Fails​

Problem: OAuth authorization flow not working.

Solutions:

  1. Verify Redirect URI
// Ensure redirect URI matches exactly (including trailing slashes)
const redirectUri = 'https://yourapp.com/callback'; // Must match registration

// Check in authorization URL
const authUrl = `https://api.metaenix.com/v1/oauth/authorize?` +
  `client_id=${clientId}&` +
  `redirect_uri=${encodeURIComponent(redirectUri)}&` +
  `response_type=code&` +
  `scope=${scopes}`;
  1. Validate State Parameter
// Always validate state to prevent CSRF
const state = generateRandomString();
sessionStorage.setItem('oauth_state', state);

// In callback
const { code, state: receivedState } = req.query;
const storedState = sessionStorage.getItem('oauth_state');

if (receivedState !== storedState) {
  throw new Error('Invalid state parameter');
}

Connection Issues​

WebSocket Connection Fails​

Problem: Cannot establish WebSocket connection.

Solutions:

  1. Check websocketId
// Verify websocketId is valid UUID format
function isValidUUID(str) {
  const uuidRegex = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;
  return uuidRegex.test(str);
}

if (!isValidUUID(websocketId)) {
  console.error('Invalid websocketId format');
}
  1. Handle Connection Errors
const socket = io('https://api.metaenix.com/account', {
  query: { websocketId },
  transports: ['websocket']
});

socket.on('connect_error', (error) => {
  console.error('Connection error:', error.message);
  
  if (error.message.includes('websocketId')) {
    // Invalid websocketId
    console.error('Check your websocketId');
  } else if (error.message.includes('timeout')) {
    // Network timeout
    console.error('Connection timeout - check network');
  }
});
  1. Check Firewall/Network
// Test WebSocket connectivity
async function testWebSocketConnection() {
  try {
    const socket = io('https://api.metaenix.com/account', {
      query: { websocketId: 'test-id' },
      timeout: 5000
    });
    
    await new Promise((resolve, reject) => {
      socket.on('connect', resolve);
      socket.on('connect_error', reject);
      setTimeout(() => reject(new Error('Timeout')), 5000);
    });
    
    console.log('WebSocket connection successful');
    socket.disconnect();
  } catch (error) {
    console.error('WebSocket connection failed:', error);
  }
}

Rate Limit Issues​

Getting 429 Errors Frequently​

Problem: Frequently hitting rate limits.

Solutions:

  1. Monitor Rate Limit Headers
function checkRateLimit(response) {
  const remaining = parseInt(response.headers.get('X-RateLimit-Remaining') || '0');
  const limit = parseInt(response.headers.get('X-RateLimit-Limit') || '0');
  
  if (remaining < limit * 0.2) {
    console.warn(`Rate limit warning: ${remaining}/${limit} remaining`);
  }
  
  return { remaining, limit };
}
  1. Implement Request Throttling
class RequestThrottler {
  constructor(requestsPerSecond = 10) {
    this.requestsPerSecond = requestsPerSecond;
    this.queue = [];
    this.processing = false;
  }
  
  async throttle(requestFn) {
    return new Promise((resolve, reject) => {
      this.queue.push({ requestFn, resolve, reject });
      this.process();
    });
  }
  
  async process() {
    if (this.processing || this.queue.length === 0) return;
    
    this.processing = true;
    const delay = 1000 / this.requestsPerSecond;
    
    while (this.queue.length > 0) {
      const { requestFn, resolve, reject } = this.queue.shift();
      
      try {
        const result = await requestFn();
        resolve(result);
      } catch (error) {
        reject(error);
      }
      
      if (this.queue.length > 0) {
        await new Promise(r => setTimeout(r, delay));
      }
    }
    
    this.processing = false;
  }
}
  1. Cache Responses
// Cache GET requests to reduce API calls
const cache = new Map();

async function cachedFetch(url, options) {
  if (options.method === 'GET') {
    const cached = cache.get(url);
    if (cached && Date.now() - cached.timestamp < 60000) {
      return cached.data;
    }
  }
  
  const response = await fetch(url, options);
  const data = await response.json();
  
  if (options.method === 'GET') {
    cache.set(url, { data, timestamp: Date.now() });
  }
  
  return data;
}

Data Issues​

Pagination Not Working​

Problem: Pagination returns incorrect results or doesn't work.

Solutions:

  1. Verify Pagination Parameters
// Ensure page is 1-based, not 0-based
const page = 1; // First page
const limit = 20; // Items per page

const response = await fetch(
  `https://api.metaenix.com/v1/posts?page=${page}&limit=${limit}`
);
  1. Handle Pagination Metadata
async function fetchPaginated(endpoint, options = {}) {
  const results = [];
  let page = 1;
  let totalPages = 1;
  
  do {
    const response = await fetch(`${endpoint}?page=${page}&limit=100`, options);
    const data = await response.json();
    
    results.push(...data.data);
    totalPages = data.meta.totalPages;
    page++;
    
    // Small delay to avoid rate limits
    if (page <= totalPages) {
      await new Promise(resolve => setTimeout(resolve, 100));
    }
  } while (page <= totalPages);
  
  return results;
}

File Upload Fails​

Problem: File uploads failing or timing out.

Solutions:

  1. Check File Size
function validateFileSize(file, maxSizeMB = 50) {
  const maxSizeBytes = maxSizeMB * 1024 * 1024;
  if (file.size > maxSizeBytes) {
    throw new Error(`File size exceeds ${maxSizeMB}MB limit`);
  }
  return true;
}
  1. Handle Upload Progress
async function uploadWithProgress(file, onProgress) {
  const formData = new FormData();
  formData.append('file', file);
  
  const xhr = new XMLHttpRequest();
  
  xhr.upload.addEventListener('progress', (e) => {
    if (e.lengthComputable) {
      const percentComplete = (e.loaded / e.total) * 100;
      onProgress(percentComplete);
    }
  });
  
  return new Promise((resolve, reject) => {
    xhr.onload = () => {
      if (xhr.status === 200) {
        resolve(JSON.parse(xhr.responseText));
      } else {
        reject(new Error(`Upload failed: ${xhr.status}`));
      }
    };
    
    xhr.onerror = () => reject(new Error('Upload failed'));
    
    xhr.open('POST', 'https://api.metaenix.com/v1/storage/upload');
    xhr.setRequestHeader('Authorization', `Bearer ${accessToken}`);
    xhr.send(formData);
  });
}

Event Issues​

Events Not Received​

Problem: WebSocket events not being received.

Solutions:

  1. Verify Subscription
socket.emit('subscribe_follows', {}, (response) => {
  if (response.event === 'subscribed') {
    console.log('Successfully subscribed to:', response.room);
  } else {
    console.error('Subscription failed:', response);
  }
});
  1. Check Event Listeners
// Ensure event listeners are set up before subscribing
socket.on('new_follower', handleNewFollower);
socket.on('unfollow', handleUnfollow);

// Then subscribe
socket.emit('subscribe_follows');
  1. Handle Reconnections
socket.on('reconnect', () => {
  console.log('Reconnected - re-subscribing to events');
  socket.emit('subscribe_follows');
});

Debugging Tips​

Enable Debug Logging​

// Enable Socket.IO debug logging
localStorage.debug = 'socket.io-client:socket';

// Or for specific namespaces
localStorage.debug = 'socket.io-client:*';

Log All API Requests​

const originalFetch = window.fetch;
window.fetch = async function(...args) {
  console.log('API Request:', args[0], args[1]);
  const response = await originalFetch(...args);
  console.log('API Response:', response.status, response.statusText);
  return response;
};

Check Response Headers​

const response = await fetch(url, options);
console.log('Rate Limit:', {
  limit: response.headers.get('X-RateLimit-Limit'),
  remaining: response.headers.get('X-RateLimit-Remaining'),
  reset: response.headers.get('X-RateLimit-Reset')
});

Common Error Messages​

"Invalid websocketId"​

Solution: Verify your websocketId is correct and properly formatted as a UUID.

"Too many requests"​

Solution: Implement rate limit handling and request throttling.

"Invalid redirect_uri"​

Solution: Ensure redirect URI matches exactly what's registered in your app settings.

"Invalid scope"​

Solution: Check that requested scopes are valid and your app has permission.
