Scopes

Scopes define what permissions your application can access on behalf of users. When requesting authorization, you must specify which scopes your application needs.

Available Scopes​

User Scopes​

ScopeDescriptionAccess Level
read:userRead user profile informationRead
write:userUpdate user profile and settingsWrite
read:user:emailRead user's email addressRead

Post Scopes​

ScopeDescriptionAccess Level
read:postRead posts and commentsRead
write:postCreate, update, and delete postsWrite
read:post:privateRead private posts (user's own posts)Read
write:commentCreate and manage commentsWrite
write:reactionLike, share, and bookmark postsWrite

Storage Scopes​

ScopeDescriptionAccess Level
read:storageRead file metadataRead
write:storageUpload and delete filesWrite

Account Scopes​

ScopeDescriptionAccess Level
read:accountRead account informationRead
write:accountModify account settingsWrite
delete:accountDelete user accountWrite

Requesting Scopes​

Include scopes in your authorization URL as a space-separated list:

https://api.metaenix.com/v1/oauth/authorize?
  client_id=YOUR_CLIENT_ID
  &redirect_uri=YOUR_REDIRECT_URI
  &response_type=code
  &scope=read:user write:post read:post

Example​

const scopes = [
  'read:user',      // Read user profile
  'read:user:email', // Read email address
  'write:post',     // Create posts
  'read:post',      // Read posts
  'write:storage',  // Upload files
].join(' ');

const authUrl = `https://api.metaenix.com/v1/oauth/authorize?` +
  `client_id=${clientId}&` +
  `redirect_uri=${redirectUri}&` +
  `response_type=code&` +
  `scope=${encodeURIComponent(scopes)}`;

Scope Combinations​

Basic Read-Only Access​

For applications that only need to read public data:

read:user read:post

Content Creation App​

For applications that create content:

read:user write:post write:comment write:reaction write:storage

Full Access​

For applications that need comprehensive access:

read:user read:user:email write:user
read:post read:post:private write:post write:comment write:reaction
read:storage write:storage
read:account write:account

Scope Best Practices​

  1. Request minimal scopes - Only request what you need
  2. Explain why - Users are more likely to grant permissions if they understand why
  3. Start small - Request basic scopes first, add more as needed
  4. Respect user privacy - Don't request sensitive scopes unnecessarily

Checking Granted Scopes​

The token response includes the granted scopes:

{
  "access_token": "...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "read:user write:post read:post"
}

You can also check what scopes are associated with a token:

GET /v1/oauth/token/info
Authorization: Bearer YOUR_ACCESS_TOKEN

Response​

{
  "scopes": ["read:user", "write:post", "read:post"],
  "expires_at": "2025-01-15T12:00:00Z"
}

Scope Errors​

If you request an invalid scope, the authorization will fail:

{
  "error": "invalid_scope",
  "error_description": "The requested scope is invalid, unknown, or malformed"
}

Incremental Authorization​

Users can grant additional scopes later. If you need more permissions:

  1. Request authorization again with additional scopes
  2. Users will only be prompted for new scopes
  3. The new token will include all previously granted scopes plus new ones

Revoking Scopes​

Users can revoke scopes at any time through their account settings. Your application should handle cases where previously granted scopes are no longer available.
